شاخص SPI

SPI یک شاخص استاندارد برای پایش خشکسالی هواشناسی است.

SPI: بر پایه بارش محاسبه می‌شود.

روش محاسبه: بر اساس چارچوب SDAT انجام شده است.

خلاصه داده

برای مشاهده منبع، محدوده و دوره کالیبراسیون، یک دیتاست انتخاب کنید.

شاخص‌ها برای هر ایستگاه یا پلیگون به صورت مستقل و بر اساس دوره مرجع تاریخی محاسبه می‌شوند.
منابع

الهام‌گرفته از drought.eng.uci.edu.

Hao Z., AghaKouchak A., Nakhjiri N., Farahmand A. (2014). Global Integrated Drought Monitoring and Prediction System. Scientific Data, 1:140001. doi:10.1038/sdata.2014.1

Farahmand A., AghaKouchak A. (2015). A Generalized Framework for Deriving Nonparametric Standardized Drought Indicators. Advances in Water Resources, 76, 140-145. doi:10.1016/j.advwatres.2014.11.012

SPI-n چیست؟
  • از پنجره تجمعی n ماهه استفاده می‌کند؛ مانند ۱، ۳، ۶، ۱۲ یا مقیاس‌های دیگر.
  • مقیاس‌های کوتاه‌تر به تغییرات کوتاه‌مدت و فصلی حساس‌ترند.
  • مقیاس‌های بلندتر برای خشکسالی تجمعی، تنش هیدرولوژیک و کمبودهای پایدار مناسب‌ترند.
کلاس‌های شدت خشکسالی

SPI ≥ -۰/۵: شرایط نرمال یا مرطوب

-۰/۸ < SPI < -۰/۵: D0 - خشکی غیرعادی

-۱/۳ < SPI ≤ -۰/۸: D1 - خشکسالی متوسط

-۱/۶ < SPI ≤ -۱/۳: D2 - خشکسالی شدید

-۲/۰ < SPI ≤ -۱/۶: D3 - خشکسالی بسیار شدید

SPI ≤ -۲/۰: D4 - خشکسالی استثنایی

شاخص‌های روند

تائوی کندال: از -۱ تا +۱ تغییر می‌کند؛ مقادیر مثبت روند افزایشی و مقادیر منفی روند کاهشی را نشان می‌دهد.

مقدار P: شاخص معنی‌داری آماری است. معمولاً مقدار کمتر از ۰/۰۵ در سطح اطمینان ۹۵٪ معنی‌دار تلقی می‌شود.

دستیار خشکسالی

درباره این داشبورد

این داشبورد برای پایش خشکسالی طراحی شده و داده‌های مشاهداتی و شبکه‌ای اقلیمی را در یک محیط واحد برای نقشه، سری زمانی و تحلیل روند کنار هم قرار می‌دهد.

  • نقشه: با نگه‌داشتن نشانگر جزئیات سریع را ببینید و با کلیک، تحلیل منطقه را باز کنید.
  • شاخص‌ها: شاخص‌های استاندارد خشکسالی و متغیرهای پشتیبان هیدروکلیماتولوژی در یک رابط در دسترس هستند.
  • رنگ‌ها: کلاس‌های خشکسالی از D0 تا D4 همراه با وضعیت نرمال یا مرطوب نمایش داده می‌شوند.
  • کارایی: درخواست‌های API کش می‌شوند و صفحه بدون بارگذاری کامل به‌روزرسانی می‌شود.
  • چیدمان: طراحی واکنش‌گرا برای دسکتاپ و موبایل.
پایه داده
  • GHCN-Daily: رکوردهای بارش ایستگاهی برای پایش خشکسالی ایستگاه‌ها استفاده می‌شوند.
  • TerraClimate: متغیرهای شبکه‌ای هیدروکلیماتولوژی، زمینه اقلیمی و بیلان آب را گسترده‌تر می‌کنند.
  • پوشش: بیش از ۱۲٬۰۰۰ ایستگاه از نظر کیفیت غربال شده‌اند و سهم داده‌های مفقود در دوره انتخابی حداکثر ۱۰٪ است.
محصولات محاسبه‌شده
  • SPI: پایش خشکسالی استاندارد بر پایه بارش.
  • SPEI: پایش خشکسالی با در نظر گرفتن بارش و تقاضای اتمسفری.
  • SSI و متغیرهای همراه: شاخص‌های استاندارد و فیلدهای اقلیمی پشتیبان در صورت وجود نمایش داده می‌شوند.

چارچوب محاسباتی: شاخص‌های خشکسالی با چارچوب SDAT محاسبه می‌شوند و خلاصه روند با آزمون Mann-Kendall و شیب Sen گزارش می‌شود.

نمونه استفاده از API

آدرس پایه پیش‌فرض API: http://localhost:8000

  • GET /datasets - فهرست لایه‌های موجود
  • GET /meta?level=station - فراداده شامل بازه زمانی و شاخص‌ها
  • GET /mapdata?level=station&index=spi6&date=2020-01 - داده نقشه
  • GET /timeseries?region_id=10001&level=station&index=spi6 - سری زمانی
  • GET /kpi?region_id=10001&level=station&index=spi6&date=2020-01 - KPI و روند

نمونه کامل cURL:

# 1) دریافت دیتاست‌ها
curl "http://localhost:8000/datasets"

# 2) دریافت فراداده لایه ایستگاه
curl "http://localhost:8000/meta?level=station"

# 3) دریافت داده نقشه برای یک شاخص اقلیمی
curl "http://localhost:8000/mapdata?level=station&index=precip&date=2020-01"

# 4) دریافت سری زمانی یک منطقه
curl "http://localhost:8000/timeseries?region_id=10001&level=station&index=precip"

# 5) دریافت KPI و روند همان منطقه در یک ماه
curl "http://localhost:8000/kpi?region_id=10001&level=station&index=precip&date=2020-01"
